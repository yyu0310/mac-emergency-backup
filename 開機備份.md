# 進水筆電 — 開機備份 SOP

> MacBook Air M3（2024）進水後的緊急備份流程。
> 原則：**通電時間越短越好**。抓完最重要的資料就立刻關機。

---

## 為什麼要這樣做

- M3 MacBook 的 SSD 焊死在主機板且硬體加密，**資料只能靠原機開機取出**，無法拆硬碟外接。
- 進水腐蝕是**漸進的**，不保證機器能持續開機，應盡早備份。
- 所以：**一定要先備份**，抓完立刻關機，再送修換螢幕。

---

## 開機前檢查

- [ ] 機殼縫縫沒有明顯水珠在流（盡量乾一點再開）
- [ ] 外接硬碟在手邊
- [ ] 心裡有數：這次開機只做一件事 — 備份，做完馬上關機

---

## 備份步驟

### 1. 開機

- 按電源鍵開機。螢幕有色斑沒關係，能看到字就好。

### 2. 插上外接硬碟

- 插上外接硬碟。
- 等 Finder 出現硬碟圖示（約幾秒）。

### 3. 打開終端機

- `Command + 空白鍵` 打開 Spotlight → 輸入 `終端機` 或 `Terminal` → Enter。

### 4. （保險）確認硬碟名稱

在終端機輸入：

```
ls /Volumes
```

- 看到清單裡有你的外接硬碟名稱 → 正確，繼續下一步。
- 如果名字不一樣 → 打開 backup.sh，把 `DEST` 那行改成正確名稱再存檔。

### 5. 執行備份

```
bash ~/Downloads/backup.sh
```

- 畫面會一行一行顯示正在備份的項目。
- 看到 **`✅ 備份完成`** 和一個大小數字（例如 `12G`）就代表成功。

### 6. 確認 & 關機

- 備份存放位置：`/Volumes/YOUR_DRIVE_NAME/EmergencyBackup/mac-backup-時間戳/`
- 看到完成訊息和正常的大小數字 → **直接關機**。
- 外接硬碟是實體寫入，**不用等上傳**，關機即可。

---

## 備份內容（腳本自動處理，由最重要到次要）

1. **桌面 / VS Project** — 你的專案，最先備份
2. `~/.claude/` ＋ `~/.claude.json` — Claude Code 設定、skills、memory、歷史
3. **金鑰/設定**：`~/.ssh/`、`~/.config/`、`~/.gitconfig`、shell rc（`.zshrc`/`.bashrc`/`.zshenv`/`.p10k.zsh` 等）
4. **開發者憑證**：`~/.aws`、`~/.gnupg`、`~/.kube`、`~/.docker`、`~/.npmrc`、`~/.netrc`、`~/.terraform.d`
5. **VS Code 設定**：`~/Library/Application Support/Code/User`
6. **launchd**：`~/Library/LaunchAgents/` ＋ plist 指向的腳本 ＋ `launchctl list` 清單
7. 自訂腳本：`~/bin`、`~/.local/bin`、`~/scripts`
8. `~/Documents`、`~/Desktop`、`~/Downloads` — 工作資料
9. **環境清單(純文字)**：Homebrew、App、VS Code 擴充、npm/pip 套件、工具版本、crontab — 供日後重灌還原

---

## 出狀況怎麼辦

| 狀況 | 處理 |
|---|---|
| 說找不到外接硬碟 | 執行 `ls /Volumes` 確認硬碟有掛上、名字沒打錯 |
| 開機後螢幕全黑/無法操作 | 別硬撐，直接帶去維修店，請他們接外接螢幕備份 |
| 開機過程有異味、燙、冒煙 | **立刻關機拔電**，不要再開，直接送修 |
| 備份跑很久 | 正常，工作資料多會比較久；耐心等到 `✅ 備份完成` |

---

## 關機後

- 繼續立著、電風扇吹、保持關機。
- 確認外接硬碟裡的備份檔案都在，再放心送修。

---

## 日後還原時注意（exFAT 限制）

KINGSTON 是 exFAT 格式，不保存 Unix 權限，所以還原金鑰後要手動修權限：

```
chmod 700 ~/.ssh && chmod 600 ~/.ssh/*
chmod 700 ~/.gnupg && chmod 600 ~/.gnupg/*
```

否則 SSH 會因「權限太寬鬆」拒絕使用金鑰。GPG 同理。
其他一般檔案不受影響。
